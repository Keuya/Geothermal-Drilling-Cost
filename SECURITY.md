# Security Policy

## Reporting a vulnerability
If you discover a security issue related to this repository (including exposed credentials, unsafe scripts, or sensitive data leaks), please do not open a public issue.

Instead:
1. Open a private communication channel with the maintainer (if available), or
2. Open an issue with minimal details and request secure follow-up.

Please include:
- Description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if known)

## Response expectations
Maintainers will acknowledge reports as soon as possible and prioritize remediation based on severity.
